Create `src/settings.ts` with a `TogglImportSettingTab` class that extends `PluginSettingTab`.

Use tabs for indentation (matching src/main.ts convention).

**Imports:**
```typescript
import { App, PluginSettingTab, Setting } from 'obsidian';
import TogglImportPlugin from './main';
```

**Class structure:**
- `export class TogglImportSettingTab extends PluginSettingTab`
- Property: `plugin: TogglImportPlugin`
- Constructor: `constructor(app: App, plugin: TogglImportPlugin)` calling `super(app, plugin)` and assigning `this.plugin = plugin`

**`display()` method — render settings in this exact order:**

1. Call `containerEl.empty()` at the top (prevents duplicate rendering on re-render).

2. **API token field** (per D-03, D-04, SET-01, SET-02):
   - `new Setting(containerEl)`
   - `.setName('Toggl API token')`
   - `.setDesc('Stored in plain text in data.json \u2014 included in Obsidian Sync.')`
   - `.addText(text => text.setPlaceholder('Enter your API token').setValue(this.plugin.settings.apiToken).onChange(async (value) => { this.plugin.settings.apiToken = value; await this.plugin.saveSettings(); }))` — then chain `.then(c => { c.inputEl.type = 'password'; })` on the text component to mask the input.

3. **Output format dropdown** (per D-05, SET-03):
   - `new Setting(containerEl)`
   - `.setName('Output format')`
   - `.addDropdown(drop => drop.addOption('table', 'Markdown table').addOption('plaintext', 'Plain text').setValue(this.plugin.settings.outputFormat).onChange(async (value) => { this.plugin.settings.outputFormat = value as 'table' | 'plaintext'; this.display(); await this.plugin.saveSettings(); }))`
   - Note: `value as 'table' | 'plaintext'` cast required because DropdownComponent.onChange provides `string` (per RESEARCH.md Pitfall 1).
   - Note: `this.display()` called before `await this.plugin.saveSettings()` — mutation is synchronous so display() sees the new value (per RESEARCH.md Pitfall 2).

4. **Conditional delimiter field** (per D-06, SET-05):
   - Only render when `this.plugin.settings.outputFormat === 'plaintext'`:
   ```typescript
   if (this.plugin.settings.outputFormat === 'plaintext') {
       new Setting(containerEl)
           .setName('Delimiter')
           .setDesc('Character(s) used to separate columns in plain text output.')
           .addText(text => text
               .setValue(this.plugin.settings.delimiter)
               .onChange(async (value) => {
                   this.plugin.settings.delimiter = value;
                   await this.plugin.saveSettings();
               })
           );
   }
   ```

5. **Columns section heading** (per D-07, SET-04):
   - `new Setting(containerEl).setName('Columns').setHeading();`

6. **Five column toggles** in order: Description, Start time, Duration, Tags, Project (per D-07, SET-04):
   - Define an array of `{ key, label }` objects:
     ```typescript
     const columns: Array<{ key: keyof typeof this.plugin.settings.columns; label: string }> = [
         { key: 'description', label: 'Description' },
         { key: 'startTime', label: 'Start time' },
         { key: 'duration', label: 'Duration' },
         { key: 'tags', label: 'Tags' },
         { key: 'project', label: 'Project' },
     ];
     ```
   - Loop over the array, creating a `Setting.addToggle()` for each:
     ```typescript
     for (const col of columns) {
         new Setting(containerEl)
             .setName(col.label)
             .addToggle(toggle => toggle
                 .setValue(this.plugin.settings.columns[col.key])
                 .onChange(async (value) => {
                     this.plugin.settings.columns[col.key] = value;
                     await this.plugin.saveSettings();
                 })
             );
     }
     ```

**No validation logic** — Phase 5 handles token validation. Empty token field is silent (per CONTEXT.md Claude's Discretion and RESEARCH.md Open Questions #2).
